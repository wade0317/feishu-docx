# Markdown 文档格式要求

本文用于规范 Markdown 文档在同步到飞书文档前的标题与内容格式，确保导入后层级清晰、编号统一。

## 1. 标题编号规则

### 1.1 一级标题

- 一级标题使用 `#`。
- 标题文本必须以阿拉伯数字编号开头，格式为 `1. `、`2. `、`3. `。
- 示例：

```markdown
# 1. 文档说明
# 2. 图表类型
# 3. 附录
```

### 1.2 二级标题

- 二级标题使用 `##`。
- 编号格式为 `一级标题序号.二级标题序号`。
- 示例：

```markdown
## 1.1 文档定位
## 1.2 编写要求
## 2.1 图表速查表
```

### 1.3 三级标题

- 三级标题使用 `###`。
- 编号格式为 `一级标题序号.二级标题序号.三级标题序号`。
- 示例：

```markdown
### 1.3.1 Mermaid 推荐模板
### 2.2.1 流程图（Flowchart）
### 3.2.10 思维导图（Mindmap）
```

## 2. 层级使用要求

- 标题最多使用到三级。
- 不要跳级使用标题。
- 一级标题下如需正文，先写普通段落，再进入二级标题。
- 从文档结构表达上看，一级标题下的正文、二级标题、以及二级标题下的正文内容，都应相对一级标题缩进一个层级单位。
- 二级标题下如需继续展开三级标题，则三级标题及其正文内容应继续保持相对二级标题的下一级层级关系。
- 二级标题下的普通正文段落会在飞书中按一个缩进单位处理。
- 列表块、代码块、引用块不会自动继承正文缩进。

## 3. 内容组织要求

### 3.1 普通正文

- 正文使用普通段落，不要把正文写进标题行。
- 段落之间保留空行。

### 3.2 列表

- 无序列表使用 `-`。
- 有序列表仅用于正文步骤说明，不用于替代标题编号。

### 3.3 代码块

- 普通源码使用三反引号代码块。
- Mermaid 图表使用 ```mermaid。
- PlantUML 图表使用 ```plantuml。

## 4. 推荐模板

```markdown
# 1. 文档说明与编写规范

本文用于说明文档目标、结构和编写方式。

## 1.1 文档定位

这里是一级标题下的下一级内容。

- 第一部分说明文档编写要求
- 第二部分整理图表类型

## 1.2 文档编写要求

正文段落写在这里。

## 1.3 统一导入结构

### 1.3.1 Mermaid 推荐模板

这里是三级标题对应的说明内容。

```mermaid
graph LR
    A[开始] --> B[结束]
```
```

## 5. 禁止项

- 不要使用中文序号作为标题前缀，例如 `一、`、`二、`、`三、`。
- 不要在三级标题前再额外写一套独立编号，例如 `### 2.2.1 1. 流程图`。
- 不要把标题写成有序列表项。
- 不要混用同一层级的不同编号风格。

## 6. 同步到飞书后的预期效果

- 一级标题对应飞书一级标题。
- 二级标题对应飞书二级标题。
- 三级标题对应飞书三级标题。
- 一级标题下的正文、二级标题和二级标题下的正文，在结构上都作为一级标题的下一级内容。
- 二级标题下的普通正文段落在飞书中缩进一个单位。
